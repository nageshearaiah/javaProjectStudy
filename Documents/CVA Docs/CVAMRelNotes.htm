ClientVantage Agentless Monitoring – recommended browsers and connectivity

The following browsers are supported by ClientVantage Agentless Monitoring report servers:

  • Microsoft Internet Explorer version 6.0 or later.

  • Mozilla Firefox version 1.5.0 or later, with JavaScript and cookie support enabled.

  • Other browsers with support for cookies, JavaScript, and CSS 2 may also be used, but they are not recommended.

Note: 

Without JavaScript enabled, the top menu of the report server will not be visible and you will see the following message instead: This product uses JavaScript. Please make sure JavaScript is enabled in your browser settings.

To enable JavaScript in Microsoft Internet Explorer, select Tools → Internet Options → Security from the top menu in your browser. Next, choose the Custom level button and enable the Active Scripting on the list of options.

To enable JavaScript in Mozilla Firefox, select Tools → Options → Content from the top menu in your browser, and select the Enable JavaScript check box.

Configuration screens require Web browsers with Java plugin version 1.5.0.9 or higher.

ClientVantage Agentless Monitoring – recommended server systems

The following server systems are officially recommended for the ClientVantage Agentless Monitoring suite:

  • Microsoft Windows Server 2003 (64-bit Edition) with latest release, service packs

  • Microsoft Windows Server 2008 (64-bit Edition) with latest release, service packs

  • Microsoft SQL 2005 Server (64-bit Edition)

  • Microsoft SQL 2008 Server (64-bit Edition)

3. What's New

Cisco Network Analysis Module as measurement data source for Vantage Analysis Server

VAS 11.1 can collect measurement data from multiple Cisco NAMs, and NAM data can be supplemented with AMD measurements to extend application performance analysis into transaction and user name context for critical applications on the network.

End-user experience monitoring in SAP R/3 environment

The SAP analyzer measures interactions of the SAP GUI clients with the SAP servers using the passive agentless network monitoring technology and network protocol analysis. All activity and user names are decoded from the traffic exchanged between clients and servers. The current release supports only Dialog-level analysis (not RFC-level analysis) for SAP GUI monitoring.

The end-to-end SAP infrastructure monitoring enables CVAM to provide a complete picture of SAP application end-user experience, and helps to diagnose problems.

User-level response monitoring for Web applications

Content delivered to end users by modern, complex Web applications is built by multiple tiers of the application infrastructure – a mesh of application servers, Web services, messaging services, and databases. The only place where all this content is put together and becomes visible is the end user's browser. AMD analyzes and categorizes the messages returned by the Web application, enabling Vantage to diagnose which end-user actions cause specific responses or errors, and reports precisely on messages that end users have seen on their screens. The response analysis is expanded by support for custom transaction metrics, which are calculated based on data requested by end users.

Currency updates

  • HTTP analyzer: user recognition in Siteminder environment

  • Oracle Forms analyzer: user name tracking between Oracle server sockets

  • Duplicate packet resolution in presence of some network devices

  • Fixes to the Cross-Site Scripting vulnerability on VAS

4. Technical Notes

CryptoSwift Accelerator Support

Starting with version 11.1, ClientVantage Agentless Monitoring does not support the CryptoSwift Accelerator Card. The migration procedure is described in the CVAM – Migrating SSL Acceleration from Discontinued SafeNet CryptoSwifttechnical paper published on FrontLine (http://frontline.compuware.com).

Support for AMD OS

Primary AMD operating system is now Red Hat Enterprise Linux. Although AMD 11.1 works on both AMD OS 3.6 and RHEL, there are limitations in support of certain network cards on AMD OS. AMD release 11.1 is the last AMD release that supports AMD OS. Starting from next release after 11.1 RHEL will be the only supported AMD operating system.

Microsoft Internet Explorer performance

Due to a different handling of the data within the HTML, Microsoft Internet Explorer may experience degradation in performance while viewing reports containing a large number of columns or reports containing a large number of tooltips.

Discrepancies between AWDS and VAS.

Starting with release 11.1, the number of pages and hits reported by the HTTP decode may, in certain cases, be different from in previous releases. This difference will happen in cases when URL autolearning was enabled and certain number of non-HTML URLs were calculated based on their HTML referrer, which in fact was already counted when the HTML was first observed. The URL count may therefore be slightly exaggerated.

You could configure a parameter to disable the URL learning option and eliminate the possibility of an inaccurate count, but starting with Vantage version release 11.1 this option is built in and there is no configuration necessary. As a result, the HTML referrer URL will be compared to learned URLs and matched URLs counted as one hit, making the URL hit count more precise.

Expected Differences per analyzer between previous and current ClientVantage Agentless Monitoring versions.

HTTP analyzer compared to version 11.1:

  • Application errors

    In previous releases of CVAM, for each URL / operation only one application error with the highest severity was taken into account. CVAM 11.1 reports all application responses that meet the defined pattern.

    Note that in CVAM 11.1, the application error reporting mechanism has been substituted with an enhanced, universal solution: reporting on application responses.

00000 12895

If AMD and Cisco NAM monitor the same traffic and the same VAS receives data for software services from these devices, the mechanism for deduplication built in to the report server will not work.

Be aware that the workaround for such cases is still experimental. This workaround has not yet been thoroughly tested.

Use NAM_AMD_DEDUPLICATION_VALID_TIME, which you can access by typing http://VAS_ADDRESS/userpropertiesadmin in your Web browser's address bar.

This property sets the number of minutes for which the server will retain a record of those software service sessions that have been reported by AMD devices. During this time, matching sessions reported by Cisco NAM devices will be marked as duplicate and discarded.

It is recommended that you set this feature to three times the size of the monitoring interval (in minutes).

A value of zero disables the deduplication mechanism. You must restart the server to put your changes into effect.

DT 00000 12961

VCAEUE Console application might crash with a No More Handles error if the user has about 30 to 40 other applications open at the same time.

This problem occurs mainly on a Windows XP system that has an NT User Handles limit (http://support.microsoft.com/kb/327699). As a result, applications that require many NT User handles may occasionally stop working or crash.

It is recommended not to use or open a large number of applications at the same time when VCAEUE Console is running.

If you experience a No More Handles error, close all running programs on your machine and restart VCAEUE Console.

6.1. Getting Help

When calling, please provide Customer Support with as much information as possible about your environment and the circumstances that led to the difficulty. You should be ready to provide:

  • Client number: this number is assigned to you by Compuware and is recorded on your sales contract.

  • The version number of the Agentless Monitoring Device (AMD) and the report servers.

    For the report server

    Use the report server GUI by selecting Help → Product Information → About, or Tools → Diagnostics → System Status.

    For the AMD

    Scroll down to the Testing AMD section. At the bottom of the diagnostic data paragraph, look for “Version ND-RTM v.ndw.x.yy.zz”.

  • Environment information, such as the operating system and release (including service pack level) on which the product (AMD, report server) is installed, memory, hardware/network specifications, and the names and releases of other applications that were running.

    Problem description, including screenshots.

  • Exact error messages, if any (screenshots recommended).

  • Whether or not the problem is reproducible. If yes, include a sequence of steps for problem recreation. If not, include a description of the actions taken before the problem occurred.

  • A description of the actions that may have been taken to recover from the difficulty and their results.
